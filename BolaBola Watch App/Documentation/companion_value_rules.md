# BolaBola 陪伴值计算规则

## 数值基础

| 项目 | 数值 |
|------|------|
| companionValue 范围 | 0 ~ 100 |
| 初始值 | 50 |

---

## 一、互动刷新规则

**产品语义**：陪伴值应该给用户持续互动的动力。用户有任何有效互动时，刷新“最后互动时间”，在接下来 1 小时内不扣分。

**有效互动包括**：
- 点击 Bola
- 语音 / 聊天
- 喂食
- 喝水
- 睡觉 / 叫醒
- iPhone 端触发并同步到 Watch 的同类互动

**实现要点**：
- 持久化 **`lastCompanionInteractionWallClock`**：最后一次有效互动的 Unix 时间戳。
- 持久化 **`lastCompanionWallClock`**：上次结算未互动扣分 / 惊喜累计的墙钟时刻。
- 每次有效互动都会同时刷新这两个时间戳，扣分从新的互动时间重新开始计算。
- `companionValue` 不再靠打开 App 或墙钟挂机被动加分；上升主要来自点击、喂食等奖励。

---

## 二、扣分规则（未互动自动下降）

当 **当前时刻距离最后一次有效互动超过 1 小时** 后，开始自动扣分：

- 前 **1 小时** 是宽限期，不扣分。
- 宽限期后，清醒未互动时间约 **每小时 -5 分**。
- 每天 **23:30 ~ 次日 07:00** 视为睡眠时段，不参与扣分。
- 最终仍受 **0～100** 的 `companionValue` 范围约束。

示例：
- 12:00 互动，12:00~13:00 不扣分。
- 12:00 互动，15:00 再打开：超过宽限 2 小时，约扣 10 分。
- 23:30 入睡到 07:00：睡眠时段不扣分。
- 23:00 最后互动，08:00 再打开：只计算 23:00~23:30 和 07:00~08:00 的清醒未互动时间，扣分很轻。

---

## 三、参数汇总

| 参数 | 数值 |
|------|------|
| 互动后扣分宽限期 | 1 小时 |
| 扣分速率 | 约 -5 分 / 小时 |
| 睡眠豁免时段 | 23:30 ~ 07:00 |
| 单次扣分上限 | 无，最低 0 分 |
| 被动加分 | 无 |

---

## 四、实现说明（代码）

- `PetViewModel`：`lastCompanionInteractionTime`、`interactionDecayGraceSeconds`、`companionDecayPerHour`、`applyInteractionDecayIfNeeded`。
- `CompanionPersistenceKeys.lastCompanionInteractionWallClock`：跨启动、跨 Watch/iPhone 同步最后互动时间。
- `lastCompanionWallClock` 仍用于“上次结算点”，避免同一段未互动时间重复扣分。
