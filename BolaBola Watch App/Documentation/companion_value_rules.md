# BolaBola 陪伴值计算规则

## 数值基础

| 项目 | 数值 |
|------|------|
| companionValue 范围 | 0 ~ 100 |
| 初始值 | 50 |

---

## 一、加分规则（会话墙钟）

**产品语义**：用户打开 BolaBola 后，在**未出现「长期离线」**的前提下，垂腕、手表睡眠、App 挂起期间的**墙钟时间**仍计入陪伴（不必常亮屏幕）。

**实现要点**：
- 持久化 **`lastCompanionWallClock`**：上次把墙钟间隔结算进陪伴分/惊喜累计的时刻。
- **不**在进后台、睡眠时推进该时间戳；在 **回到前台**、**Timer 每 60 秒**、**冷启动 hydrate** 时，用 **`当前时刻 − lastCompanionWallClock`** 结算。
- 若间隔 **≤ `longAbsenceWithoutForegroundSeconds`（默认 24 小时）**：整段计入加分（每满 `secondsPerCompanionBonus` 秒 +1，**无每日加分上限**；**当前代码为 600 秒 = 10 分钟 +1，便于测试；正式可改回 3600 = 每小时 +1**）。
- 总分仍受 **0～100** 的 `companionValue` 范围约束。

**时段**：全天 24 小时均可加分。

---

## 二、扣分规则（自动检测「长期离线」）

**无需用户点按钮**。当 **`当前时刻 − lastCompanionWallClock`** **大于** **`longAbsenceWithoutForegroundSeconds`**（默认 **24 小时**）时，视为用户**长期未回到 App**：

- **不**把这段超长间隔按挂机加分；
- 对 **「上次打点时刻 → 本次结算时刻」** 按 Gap **扣分**（剔除 00:00～07:00、2 小时宽限、每 5 分钟 −0.1）；**离线越久，理论扣分越多**（无「单次最多 −8」封顶，最终仍不低于 0 分）。

**阈值说明**：24 小时内再打开，仍按第一节整段墙钟加分（含睡眠）；超过 24 小时才走本节。可在代码中调整 `longAbsenceWithoutForegroundSeconds`。

---

## 三、参数汇总

| 参数 | 数值 |
|------|------|
| 加分速率 | +1 分 / 小时（墙钟间隔折算） |
| 每日加分上限 | **无**（仅受 0～100 总分限制） |
| 长期离线阈值 | 默认 **24 小时**（自动比较墙钟间隔） |
| 扣分宽限期 | 2 小时（仅长期离线分支内的有效 Gap） |
| 扣分速率 | -0.1 分 / 5分钟 |
| 夜间豁免时段（扣分） | 00:00 ~ 07:00 |
| 单次扣分上限 | **无**（随有效 Gap 增大；最低 0 分） |

---

## 四、示例

### 加分

- 打开后工作几小时、中间手表睡眠，**24 小时内**再进 App：中间整段按墙钟累计。

### 扣分（自动）

- **超过 24 小时**才再次打开：触发「长期离线」分支，按 Gap 可能扣分，且**不会**把整段离线当挂机加分。
