# 特殊动画图鉴

> 更新时间：2026-04-28
> 目标：沉淀 Bola 特殊动画的触发主题、解锁条件与实现落点，便于后续资源制作与状态机接入。

## 文档定位

这份文档用于整理 `常驻情绪动画` 之外的 `特殊动画 / 彩蛋动画 / 纪念性动画`。

建议后续把每个条目都补齐以下四个实现字段：

- 动画资源名：对应 `Assets.xcassets` 中的前缀或目录
- Emotion case：对应 `PetEmotion`
- 触发入口：日期、行为、语义、健康、回归等来源
- 播放策略：一次性播放 / 可重复 / 当日限一次 / 解锁后常驻图鉴

## 建议统一规则

- 特殊动画优先定义为 `play-once`，播完回到默认态或轻情绪态。
- 节日类动画建议 `当日限一次`，避免重复打扰。
- 纪念类动画建议记录 `首次触发时间`，用于图鉴点亮与回顾。
- 生活作息、互动行为、情绪语义类动画建议允许重复触发，但要加冷却时间。
- 图鉴侧建议保存：
  `animationID`、`firstUnlockedAt`、`lastPlayedAt`、`playCount`、`category`。

## 私人纪念日

| 触发点 | 触发条件 | 实现备注 |
|--------|----------|----------|
| 初遇纪念日 | 用户首次创建 / 首次开始使用 Bola 的当天 | 需要首日时间戳 |
| 初遇周年 | 用户首次使用满 365 天后打开 App | 可按周年持续复用 |
| 陪伴值首次达到 100 | 陪伴值第一次达到 100 时解锁 | 只应触发一次 |
| 第一次连续陪伴 7 天 | 连续 7 天每天至少与 Bola 互动 1 次 | 依赖连续活跃统计 |

## 图鉴字段建议

如果后面要做“特殊动画图鉴”功能，建议每个动画至少有以下元数据：

| 字段 | 含义 |
|------|------|
| `id` | 唯一标识，例如 `festival_mid_autumn_2026` |
| `title` | 图鉴显示名 |
| `category` | `festival` / `anniversary` / `routine` / `interaction` / `semantic` / `health` / `returning` |
| `unlockRule` | 触发条件描述 |
| `isHiddenBeforeUnlock` | 未解锁前是否隐藏 |
| `firstUnlockedAt` | 首次解锁时间 |
| `lastPlayedAt` | 最近播放时间 |
| `playCount` | 累计播放次数 |
| `animationEmotion` | 对应 `PetEmotion` 或动画组 |
| `rewardText` | 解锁文案 / 回忆文案 |

## 实现拆分建议

建议按下面顺序推进，避免一次把所有特殊动画逻辑塞进主状态机：

1. 先做 `节日类 + 私人纪念日`，因为判定最清晰、最适合做图鉴 MVP。
2. 再接 `互动行为 + 情绪语义`，这两类能强化陪伴感，但需要事件埋点与语义标签。
3. 最后接 `健康状态 + 回归状态 + 世界主题日`，因为它们依赖更多跨模块数据。

## 与代码结构的对应关系

- 动画资源与映射：`BolaBola Watch App/Pet/PetAnimation.swift`
- 播放状态机与触发入口：`BolaBola Watch App/Views/ContentView.swift`
- 对话文案池：`BolaBola Watch App/Pet/BolaDialogueLines.swift`
- 健康与行为数据来源：`Shared/`
- 后续图鉴持久化：建议放在 `Shared/` 下，保证 iPhone / Watch / Widget 可复用
